📋 System Guidelines & Security Notice
⚠️

RBAC Not Configured — Security Risk

Without Role-Based Access Control, any user can push directly to any branch. This exposes your repositories to unauthorized changes and data leaks.

🔓

Branch Protection Required

Without branch protection on main, developers can commit directly — risking broken production deployments. Always enable PR-only merges.

✅

What Happens After Execution

For each service: a private Gitea repository is created, code is pushed from your local branch to main, and status is updated in real time.

🔐

Admin-Only Push Policy

Only the Admin account may push directly to main. All developers must open a Pull Request for review before any merge is allowed.
